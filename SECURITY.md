# Security Policy

AuthLens는 허가된 시스템의 인증 흐름을 안전하게 관찰·문서화하기 위한 도구입니다. 우리는 도구가 오용되는 것을 막기 위해 보수적인 기본값을 유지합니다.

## 안전 기본값 (Safe Defaults)

- 비밀번호, 토큰, 쿠키의 원문(raw)은 기본적으로 저장하지 않습니다.
- UI와 export에는 마스킹된 값만 노출됩니다.
- raw 표시 또는 raw export는 명시적으로 켜야 하며, 켜는 순간 경고가 표시됩니다.
- request/response body는 크기가 제한되며, binary 응답 body는 저장하지 않습니다.
- AuthLens는 자동 로그인, brute force, CAPTCHA 우회, MFA 우회를 구현하지 않습니다.
- Replay 기능은 기본 비활성이며, 활성화하더라도 원래 도메인에 자동 반복 호출하지 않습니다.

## 보고하기 (Reporting a Vulnerability)

이 프로젝트에서 보안 취약점을 발견하셨다면 공개 issue 대신 비공개로 알려주세요.

- email: `sejoung@gmail.com`
- GitHub Security Advisories를 사용하셔도 됩니다.

다음 정보를 함께 보내주시면 분석이 빠릅니다.

- 영향 받는 모듈/버전
- 재현 절차
- 가능한 경우, 영향 범위와 완화 방안

48시간 내 1차 확인을 목표로 합니다. 패치가 준비되기 전까지는 공개 노출을 자제해 주세요.

## 사용 정책 (Acceptable Use)

AuthLens는 다음 목적의 사용을 의도합니다.

- 자사/사내 서비스의 인증 흐름 디버깅
- QA, 문서화, 교육 자료 제작
- 본인이 운영하거나 명시적으로 허가받은 시스템 분석

다음은 명시적으로 권장하지 않습니다.

- 제3자 서비스에 대한 무단 분석
- 침투 테스트 자동화 또는 공격 도구로의 활용
- 타인의 세션·토큰을 탈취·재사용하는 시도
- 대량 계정 자동 로그인 또는 인증 우회
